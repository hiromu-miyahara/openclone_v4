# OpenClone UI/UX設計書（オンボーディング / メイン画面）

## 1. 目的
本書は、OpenCloneのユーザー体験を「ドット絵の世界観」で統一するために、オンボーディング画面とメイン画面のUI/UX仕様を定義する。

## 2. 体験コンセプト
- テーマ: 「暗闇から誕生し、世界に現れる分身」
- 視覚トーン:
  - 背景はほぼ黒（闇）
  - テキストは白ドット絵風
  - 光（出口）は進行に合わせて強くなる
- 体験軸:
  - 入力するほど世界が開く（進捗の物語化）
  - 完了時に「誕生演出」を提示し、メイン画面へ遷移

## 3. デザインシステム（最小）
### 3.1 カラー
- `--bg-primary: #050505`
- `--bg-secondary: #0d0d0d`
- `--text-primary: #f5f5f5`
- `--text-muted: #b8b8b8`
- `--light-core: #ffffff`
- `--light-glow: #d9e8ff`
- `--accent-danger: #ff5c5c`

### 3.2 タイポグラフィ
- 見出し: ドット絵フォント（8-bit系）
- 本文: 可読性重視の等幅または角ゴシック
- 原則: UI上の主要コピーは短文。1ブロック2行以内。

### 3.3 モーション
- 基本fps: 30fps（UI遷移）
- ドット絵アニメ: 8fps（キャラ）
- 演出:
  - フェード + 微小拡大
  - 光源の脈動
  - テキストのタイプライター表示

## 4. オンボーディング画面設計
### 4.1 ステージ構成
1. `stage-0` 暗闇の導入
- 黒背景のみ
- 遠方中央に小さな白い光点
- 白ドット文字: 「ここから、あなたの分身を生み出します」

2. `stage-1` 基本情報入力
- 名前入力
- 顔写真アップロード
- 光がわずかに広がる

3. `stage-2` 音声質問フェーズ（5〜8問）
- 音声回答必須
- 質問セットは固定（MVP中は分岐質問なし）
- 1問回答ごとに光の強度とサイズを段階増加
- 進捗UI: 「Q3/8」 + 光の到達ゲージ

4. `stage-3` 生成待機フェーズ
- 声質学習（ElevenLabs）と2等身アバター生成（B方式）を実行
- テキスト: 「いま、あなたの分身が声と姿を得ています」
- 光が出口形状に明確化
- 20秒経過でメイン画面の先行開始を許可
- 未完了時は誕生ストーリー演出をループ表示

5. `stage-4` 誕生演出
- 白い出口に到達
- 短い物語テキストを表示:
  - 例: 「あなたの分身は、この世界に生まれました」
- 完了ボタン: 「会話をはじめる」

### 4.2 入力要件（オンボーディング）
- 名前: 必須
- 顔写真: 必須（1枚）
- 音声回答: 各質問で必須
- 顔写真品質ゲート: 顔向き・明るさ・顔サイズ・解像度を満たすこと
- エラー時:
  - 録音失敗: 再録音導線
  - 写真不備: 再アップロード導線

### 4.3 進行ルール
- ユーザーが完了条件を満たすまで次ステージ不可
- 離脱復帰時は直前の質問から再開
- 完了時に人格プロファイルを確定（MVP中は自動更新なし）

## 5. メイン画面設計
### 5.1 レイアウト
- 上部:
  - 2等身ドット絵アバター表示領域
  - アクション連動（idle / thinking / speaking / nod / agree / surprised / emphasis）
- 中央:
  - 会話ログ（白文字主体）
- 下部:
  - テキスト入力欄
  - 音声入力ボタン
  - 音声再生ボタン（TTSはユーザー選択時のみ）
  - 初期モードはテキスト入力

### 5.2 会話インタラクション
1. ユーザー入力（音声 or テキスト）
2. STT（音声時）でテキスト化
3. 応答受信（`text` + `action`）
4. テキスト表示と同時にドット絵アクション再生
5. ユーザーが音声再生を押した場合のみTTS生成・再生

### 5.3 反応演出
- `thinking`: まばたき + 微振動
- `speaking`: 口パクループ
- `agree`: うなずき
- `surprised`: 上下1フレーム揺れ + 一瞬の輝度上昇
- `emphasis`: 1回の前傾動作

## 6. 状態遷移（要点）
- `OnboardingInProgress` -> `GeneratingAssets` -> `Born` -> `ChatReady`
- `ChatReady` 内状態:
  - `Idle`
  - `Listening`
  - `Thinking`
  - `Responding`
  - `PlayingVoice`

## 7. コピーガイド（抜粋）
- 導入: 「暗闇の先で、あなたの分身が目覚めます」
- 質問開始: 「あなたの声で答えてください」
- 生成中: 「声と姿を結びつけています」
- 完了: 「準備ができました。会話を始めましょう」

## 8. MVP実装優先順位（UI/UX）
1. オンボーディングのステージ遷移（暗闇→光）
2. 音声回答必須フロー
3. 生成待機・完了演出
4. メイン画面の`text + action`連動
5. 微演出（光脈動、タイプライター、画面遷移）

## 9. 非機能・品質観点
- モバイル優先（iOS Safari / Android Chrome）
- 初回ロード軽量化:
  - フォントと画像を段階ロード
- 視認性:
  - 黒背景上で白文字のコントラスト確保
- 長時間利用:
  - 点滅演出は低頻度に制限
